Experiment Notes

Note I: Experiment Introduction

‚ÄúGibbs sampling with people‚Äù (GSP) is an experimental method that embeds the idea of Gibbs sampling into the process of human participants doing experiments with the aim of learning human preferences. The method is presented by sequentially providing participants with a slider that manipulates the corresponding parameter and asking them to adjust the slider to select the value that best matches a given criterion. In our case, we innovatively applies GSP to understand how people feel about different combinations of web parameters under a criteria "readability".

Note II: Web Parameters

Line width, title font size, content font size, line height, font color, background color are selected for this experiment. The range of line width is 800-1000px. The range of title font size is 12-45px while the range of content font size id 12-30px. The range of line height is 1-2. This value is unitless and the actual distance is specified by multiplying this value by the font size of the element itself Particularly, both font colour and background color are parameterised using the HSL color system, as this color system better reflects human perception of color relationships. The system encodes each color as three integers: hue, saturation and lightness. Hue is a degree on the color wheel from 0 to 360. 0 is red, 120 is green, 240 is blue. Saturation can be described as the intensity of a color. 100% is pure color, no shades of gray; 50% is 50% gray, but you can still see the color; 0% is completely gray, you can no longer see the color. The lightness of a color can be described as how much light you want to give the color, where 0% means no light (black), 50% means 50% light (neither dark nor light) 100% means full lightness (white). In total, there are 10 web parameters that need to be manipulated.

Note III: Manipulating the slider

When you enter the experiment screen, you can see a control panel hovering in the middle. Below this is the text to be rendered. As you move the slider, the corresponding changes are displayed on the left-hand side of the content below. What you need to do is, for each web parameter, adjust the slider and choose a value that you think will maximise the readability of the web page. Once you have determined your choice for the current parameter value, click on the Save button and you will be taken to the choice for the next parameter value. To ensure that participants are not experimenting perfunctorily, we detect the event of the user moving the slider and if no movement event is detected, the web page will pop up with a prompt. This means that even if the default random value has met your choice, you still need to move the slider and locate it to the position you think is most appropriate, sorry for the inconvenience. To exclude the control panel itself from interfering with the user's evaluation of the readability of the web page, the user can drag the control panel to wherever they see fit. And the size of control panel can be minimized by clicking on the minimize button in the top left corner, which will appear as an icon in the top right corner of the page and can be restored by clicking on the icon. It is important to note that the experiment will run for a total of 20 iterations. An iteration ends when you have finished selecting the values for each set of 8 parameters. The number of remaining iterations is displayed on the operation panel for your reference. After 20 iterations, when you click the "save" button, your data will be recorded in the database and the experiment is completed.

Note IV: Important special notes

  • To ensure that the experimental data collected is valid, please read the experimental requirements carefully
  • Please don't fabricate data
  • If you are indecisive about more than one values, go with your gut feeling and choose the one you feel most comfortable with
  • Do not refresh the web page during the experiment
  • Do not click the back button during the experiment
  • Do not change the url in the address bar during the experiment
  • Please do not quit in the middle of the experiment
